# WireGuard VPN Manager

## What This Does

Automates the entire WireGuard VPN lifecycle — install the software, generate cryptographic keys, configure server and client tunnels, manage peers, and handle firewall rules. No manual editing of config files or memorizing `wg` commands.

**Example:** "Set up a WireGuard server on my VPS, add 5 client peers, generate QR codes for mobile devices, enable kill-switch."

## Quick Start (5 minutes)

### 1. Install WireGuard

```bash
bash scripts/install.sh
```

This auto-detects your OS (Ubuntu/Debian/Fedora/Arch/Alpine) and installs WireGuard tools + kernel module.

### 2. Initialize a Server

```bash
bash scripts/wg-manager.sh init-server \
  --interface wg0 \
  --address 10.0.0.1/24 \
  --port 51820 \
  --endpoint your-server-ip
```

**Output:**
```
✅ Generated server keypair
✅ Created /etc/wireguard/wg0.conf
✅ Enabled IP forwarding
✅ Added iptables rules for NAT
✅ Started wg0 interface
Server public key: aB3dE5fG7hI9jK1lM3nO5pQ7rS9tU1vW3xY5zA7bC=
```

### 3. Add a Client Peer

```bash
bash scripts/wg-manager.sh add-peer \
  --interface wg0 \
  --name "laptop" \
  --dns 1.1.1.1
```

**Output:**
```
✅ Generated peer keypair for "laptop"
✅ Added peer to wg0
✅ Client config saved to /etc/wireguard/clients/laptop.conf
✅ QR code saved to /etc/wireguard/clients/laptop.png

Client config:
[Interface]
PrivateKey = <generated>
Address = 10.0.0.2/32
DNS = 1.1.1.1

[Peer]
PublicKey = <server-pubkey>
Endpoint = your-server-ip:51820
AllowedIPs = 0.0.0.0/0
PersistentKeepalive = 25
```

## Core Workflows

### Workflow 1: Full VPN Server Setup

**Use case:** Set up a VPN server on a fresh VPS

```bash
# Install
bash scripts/install.sh

# Initialize server
bash scripts/wg-manager.sh init-server \
  --interface wg0 \
  --address 10.0.0.1/24 \
  --port 51820 \
  --endpoint $(curl -s ifconfig.me)

# Add peers
bash scripts/wg-manager.sh add-peer --interface wg0 --name "phone" --dns 1.1.1.1
bash scripts/wg-manager.sh add-peer --interface wg0 --name "laptop" --dns 1.1.1.1
bash scripts/wg-manager.sh add-peer --interface wg0 --name "tablet" --dns 1.1.1.1

# List peers
bash scripts/wg-manager.sh list-peers --interface wg0
```

### Workflow 2: Site-to-Site Tunnel

**Use case:** Connect two servers securely

```bash
# On Server A
bash scripts/wg-manager.sh init-server \
  --interface wg0 \
  --address 10.0.0.1/24 \
  --port 51820 \
  --endpoint server-a-ip

# On Server B — generate config for site-to-site
bash scripts/wg-manager.sh site-to-site \
  --interface wg0 \
  --local-address 10.0.0.2/24 \
  --remote-endpoint server-a-ip:51820 \
  --remote-pubkey "<server-a-pubkey>" \
  --allowed-ips 10.0.0.0/24,192.168.1.0/24
```

### Workflow 3: Manage Peers

```bash
# List all peers with transfer stats
bash scripts/wg-manager.sh list-peers --interface wg0

# Output:
# Name       | Public Key (short) | IP Address | Last Handshake | Transfer
# -----------|-------------------|------------|----------------|----------
# laptop     | aB3d...7bC=       | 10.0.0.2   | 2 min ago      | 1.2 GB ↓ 340 MB ↑
# phone      | xY5z...9tU=       | 10.0.0.3   | 15 min ago     | 450 MB ↓ 89 MB ↑

# Remove a peer
bash scripts/wg-manager.sh remove-peer --interface wg0 --name "tablet"

# Regenerate a peer's keys
bash scripts/wg-manager.sh regen-peer --interface wg0 --name "laptop"
```

### Workflow 4: Status & Diagnostics

```bash
# Check interface status
bash scripts/wg-manager.sh status --interface wg0

# Output:
# Interface: wg0
# Status: UP
# Public Key: aB3dE5fG7hI9jK1lM3nO5pQ7rS9tU1vW3xY5zA7bC=
# Listen Port: 51820
# Address: 10.0.0.1/24
# Peers: 3 connected, 0 disconnected
# Total Transfer: 2.1 GB received, 780 MB sent

# Test connectivity to a peer
bash scripts/wg-manager.sh ping-peer --interface wg0 --name "laptop"
```

## Configuration

### Server Config Template

Generated at `/etc/wireguard/wg0.conf`:

```ini
[Interface]
PrivateKey = <auto-generated>
Address = 10.0.0.1/24
ListenPort = 51820
PostUp = iptables -A FORWARD -i %i -j ACCEPT; iptables -t nat -A POSTROUTING -o eth0 -j MASQUERADE
PostDown = iptables -D FORWARD -i %i -j ACCEPT; iptables -t nat -D POSTROUTING -o eth0 -j MASQUERADE

[Peer]
# laptop
PublicKey = <peer-pubkey>
AllowedIPs = 10.0.0.2/32
```

### Environment Variables

```bash
# Override defaults
export WG_DEFAULT_PORT=51820
export WG_DEFAULT_DNS="1.1.1.1,1.0.0.1"
export WG_DEFAULT_SUBNET="10.0.0.0/24"
export WG_CONFIG_DIR="/etc/wireguard"
export WG_CLIENT_DIR="/etc/wireguard/clients"
```

## Advanced Usage

### Auto-Start on Boot

```bash
bash scripts/wg-manager.sh enable-boot --interface wg0
# Enables systemd: wg-quick@wg0
```

### Kill Switch (Block Non-VPN Traffic)

```bash
bash scripts/wg-manager.sh add-peer \
  --interface wg0 \
  --name "secure-laptop" \
  --dns 1.1.1.1 \
  --kill-switch
```

Adds firewall rules to client config that block all traffic except through the VPN tunnel.

### QR Code for Mobile

```bash
bash scripts/wg-manager.sh qr --interface wg0 --name "phone"
# Displays QR code in terminal (requires qrencode)
# Also saves PNG to /etc/wireguard/clients/phone.png
```

### Backup & Restore

```bash
# Backup all configs
bash scripts/wg-manager.sh backup --output /backup/wireguard-$(date +%F).tar.gz

# Restore
bash scripts/wg-manager.sh restore --input /backup/wireguard-2026-02-22.tar.gz
```

## Troubleshooting

### Issue: "RTNETLINK answers: Operation not permitted"

**Fix:** Run with sudo — WireGuard requires root for interface management.
```bash
sudo bash scripts/wg-manager.sh init-server ...
```

### Issue: Peers can't connect

**Check:**
1. Port is open: `sudo ufw allow 51820/udp` or `sudo iptables -A INPUT -p udp --dport 51820 -j ACCEPT`
2. IP forwarding enabled: `sysctl net.ipv4.ip_forward` should be `1`
3. Correct endpoint IP: `curl -s ifconfig.me`
4. Keys match: server's public key must be in client config and vice versa

### Issue: No internet through VPN

**Fix:** NAT/masquerade not configured. Check PostUp rules:
```bash
# Find your outbound interface
ip route | grep default | awk '{print $5}'
# Use that instead of eth0 in PostUp/PostDown
```

### Issue: "wireguard" kernel module not found

**Fix:**
```bash
# Ubuntu/Debian (kernel 5.6+ has it built-in)
sudo apt install wireguard-tools

# Older kernels
sudo apt install wireguard-dkms wireguard-tools
```

## Dependencies

- `wireguard-tools` (wg, wg-quick)
- `bash` (4.0+)
- `jq` (JSON peer database)
- `iptables` (NAT/firewall rules)
- `qrencode` (optional, for QR codes)
- Linux kernel 5.6+ (WireGuard built-in) or `wireguard-dkms`

## Key Principles

1. **Zero-config defaults** — Sensible subnet, port, DNS out of the box
2. **Named peers** — Track peers by name, not just public keys
3. **Automated key management** — Never manually generate or copy keys
4. **QR codes** — One scan to connect mobile devices
5. **Idempotent** — Re-running commands won't break existing config
6. **Backup-friendly** — Export/import entire VPN configuration
