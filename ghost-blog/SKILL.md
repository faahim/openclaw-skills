# Ghost Blog Manager

## What This Does

Deploys and manages [Ghost](https://ghost.org) — the professional open-source publishing platform — on your own server using Docker. Handles installation, SSL setup, theme management, backups, and updates. No Ghost(Pro) subscription needed.

**Example:** "Deploy a Ghost blog at blog.example.com with Let's Encrypt SSL, auto-backups, and a custom theme — all self-hosted."

## Quick Start (10 minutes)

### 1. Install Dependencies

```bash
bash scripts/install.sh
```

Ensures Docker and Docker Compose are installed. Skips if already present.

### 2. Deploy Ghost

```bash
bash scripts/deploy.sh --domain blog.example.com --email you@example.com
```

This creates a full Ghost stack:
- Ghost CMS (latest)
- MySQL 8 database
- Caddy reverse proxy with automatic HTTPS
- Persistent volumes for content and database

### 3. Access Your Blog

```
🌐 Blog:  https://blog.example.com
👤 Admin: https://blog.example.com/ghost
```

First visit to `/ghost` creates your admin account.

## Core Workflows

### Workflow 1: Deploy with Custom Port (No SSL)

**Use case:** Local development or testing behind another proxy.

```bash
bash scripts/deploy.sh --domain localhost --port 2368 --no-ssl
# Access at http://localhost:2368
```

### Workflow 2: Backup Everything

**Use case:** Backup content, database, themes, and images.

```bash
bash scripts/backup.sh
# Output: backups/ghost-backup-2026-03-07-225300.tar.gz
```

Backups include:
- MySQL database dump
- Ghost content directory (images, themes, settings)
- docker-compose.yml (for restoration)

### Workflow 3: Restore from Backup

```bash
bash scripts/restore.sh backups/ghost-backup-2026-03-07-225300.tar.gz
```

### Workflow 4: Update Ghost

```bash
bash scripts/update.sh
# Creates backup first, then pulls latest Ghost image
```

### Workflow 5: Install a Theme

```bash
# From GitHub
bash scripts/theme.sh install https://github.com/TryGhost/Casper

# From local zip
bash scripts/theme.sh install /path/to/theme.zip

# List installed themes
bash scripts/theme.sh list

# Activate a theme
bash scripts/theme.sh activate casper
```

### Workflow 6: Content Management via API

```bash
# Export all content as JSON
bash scripts/content.sh export > content.json

# Import content from JSON
bash scripts/content.sh import content.json

# List recent posts
bash scripts/content.sh posts --limit 10

# Create a post from markdown
bash scripts/content.sh create --title "My Post" --file post.md --status draft
```

### Workflow 7: View Logs

```bash
bash scripts/logs.sh          # All services
bash scripts/logs.sh ghost    # Ghost only
bash scripts/logs.sh db       # MySQL only
bash scripts/logs.sh caddy    # Caddy only
```

## Configuration

### Environment Variables

Set in `.env` (created by deploy.sh):

```bash
# Required
GHOST_DOMAIN=blog.example.com
GHOST_EMAIL=you@example.com          # For SSL certs

# Database
MYSQL_ROOT_PASSWORD=<auto-generated>
MYSQL_PASSWORD=<auto-generated>

# Optional
GHOST_PORT=2368                      # Internal port
GHOST_MEMBERS=true                   # Enable memberships
GHOST_MAIL_TRANSPORT=SMTP            # Email delivery
GHOST_MAIL_HOST=smtp.gmail.com
GHOST_MAIL_PORT=587
GHOST_MAIL_USER=you@gmail.com
GHOST_MAIL_PASSWORD=app-password
```

### Docker Compose Override

For custom config, edit `docker-compose.override.yml`:

```yaml
services:
  ghost:
    environment:
      # Custom Ghost config
      mail__transport: SMTP
      mail__options__host: smtp.mailgun.org
      mail__options__port: 587
      mail__options__auth__user: postmaster@mg.example.com
      mail__options__auth__pass: your-password
```

### Scheduled Backups

```bash
bash scripts/schedule-backup.sh --interval daily --keep 7
# Keeps last 7 daily backups, runs at 3am
```

## Troubleshooting

### Issue: "port 443 already in use"

**Fix:** Another service is using HTTPS port.
```bash
# Check what's using port 443
sudo ss -tlnp | grep :443

# Deploy with custom port
bash scripts/deploy.sh --domain example.com --https-port 8443
```

### Issue: Ghost shows "Site is starting up"

**Fix:** Ghost needs time to boot. Wait 30-60 seconds. If persistent:
```bash
bash scripts/logs.sh ghost    # Check for errors
docker compose restart ghost  # Restart Ghost
```

### Issue: Email not sending

**Fix:** Configure SMTP in `.env`:
```bash
GHOST_MAIL_TRANSPORT=SMTP
GHOST_MAIL_HOST=smtp.gmail.com
GHOST_MAIL_PORT=587
GHOST_MAIL_USER=your-email@gmail.com
GHOST_MAIL_PASSWORD=your-app-password  # NOT your Gmail password
```
Then: `docker compose up -d ghost`

### Issue: Want to completely remove

```bash
bash scripts/destroy.sh
# Stops containers, removes volumes (asks for confirmation)
```

## Dependencies

- `docker` (20.10+) — container runtime
- `docker-compose` (v2+) — orchestration
- `bash` (4.0+)
- `curl` — API calls and health checks
- `openssl` — password generation
- Root/sudo access (Docker management)
- Open ports: 80, 443 (or custom)

## Architecture

```
Internet → Caddy (SSL/reverse proxy) → Ghost (Node.js) → MySQL
                                           ↓
                                    Content Volume
                                    (images, themes)
```
