# Linkding Bookmark Manager

## What This Does

Installs and manages [Linkding](https://github.com/sissbruecker/linkding), a lightweight self-hosted bookmark manager. Save, tag, search, and archive bookmarks from the command line. Includes full-text search, automatic website snapshots, bulk import/export, and a clean web UI.

**Example:** "Save 500 bookmarks with tags, search them instantly, auto-archive pages before they disappear, export to HTML anytime."

## Quick Start (5 minutes)

### 1. Install & Start Linkding

```bash
bash scripts/install.sh
```

This starts Linkding on port 9090 with a superuser account. Credentials are printed after install.

### 2. Save Your First Bookmark

```bash
# Set your API token (printed during install, or find in Linkding UI → Settings → API)
export LINKDING_URL="http://localhost:9090"
export LINKDING_TOKEN="your-api-token"

# Save a bookmark
bash scripts/linkding.sh add "https://github.com/sissbruecker/linkding" --tags "tools,bookmarks,self-hosted"
```

### 3. Search Bookmarks

```bash
bash scripts/linkding.sh search "self-hosted"
```

## Core Workflows

### Workflow 1: Add Bookmarks with Tags

```bash
# Add with tags and description
bash scripts/linkding.sh add "https://news.ycombinator.com" \
  --tags "news,tech,daily" \
  --title "Hacker News" \
  --description "Tech news aggregator"

# Add multiple bookmarks from a file (one URL per line)
bash scripts/linkding.sh bulk-add bookmarks.txt --tags "imported"
```

### Workflow 2: Search & Browse

```bash
# Full-text search
bash scripts/linkding.sh search "docker tutorial"

# List by tag
bash scripts/linkding.sh list --tag "self-hosted"

# List recent bookmarks
bash scripts/linkding.sh list --limit 20

# List all tags
bash scripts/linkding.sh tags
```

### Workflow 3: Export & Import

```bash
# Export all bookmarks to Netscape HTML (compatible with browsers)
bash scripts/linkding.sh export > bookmarks.html

# Import from browser export
bash scripts/linkding.sh import bookmarks.html
```

### Workflow 4: Archive Pages

```bash
# Enable background archiving (saves page snapshots via ld-fetch-singlefile)
bash scripts/install.sh --with-archiving

# Manually trigger archive for a bookmark
bash scripts/linkding.sh archive 42
```

### Workflow 5: Manage Linkding Service

```bash
# Check status
bash scripts/linkding.sh status

# Stop / Start / Restart
bash scripts/linkding.sh stop
bash scripts/linkding.sh start
bash scripts/linkding.sh restart

# View logs
bash scripts/linkding.sh logs

# Update to latest version
bash scripts/linkding.sh update

# Backup database
bash scripts/linkding.sh backup
# → Creates timestamped backup in ~/.linkding/backups/

# Restore from backup
bash scripts/linkding.sh restore ~/.linkding/backups/linkding-2026-03-06.db
```

## Configuration

### Environment Variables

```bash
# Required for API operations
export LINKDING_URL="http://localhost:9090"   # Linkding instance URL
export LINKDING_TOKEN="your-api-token"        # API token from Settings → API

# Optional
export LINKDING_PORT="9090"                   # Port for Docker (default: 9090)
export LINKDING_DATA="$HOME/.linkding"        # Data directory (default: ~/.linkding)
```

### Docker Compose (auto-generated by install.sh)

```yaml
# ~/.linkding/docker-compose.yml
services:
  linkding:
    image: sissbruecker/linkding:latest
    container_name: linkding
    ports:
      - "${LINKDING_PORT:-9090}:9090"
    volumes:
      - "${LINKDING_DATA:-./data}:/etc/linkding/data"
    environment:
      - LD_SUPERUSER_NAME=admin
      - LD_SUPERUSER_PASSWORD=${LINKDING_ADMIN_PASS}
    restart: unless-stopped

  # Optional: Enable for full-page archiving
  # linkding-plus:
  #   image: sissbruecker/linkding:latest-plus
```

## Advanced Usage

### Run Behind Reverse Proxy

```bash
# Install on custom port, no direct exposure
LINKDING_PORT=9090 bash scripts/install.sh

# Then configure Nginx/Caddy to proxy to localhost:9090
# Set LD_CSRF_TRUSTED_ORIGINS for your domain:
docker exec linkding env LD_CSRF_TRUSTED_ORIGINS=https://bookmarks.example.com
```

### Scheduled Backups via Cron

```bash
# Daily backup at 2 AM
0 2 * * * cd ~/.linkding && bash /path/to/scripts/linkding.sh backup >> /var/log/linkding-backup.log 2>&1
```

### API Direct Access

```bash
# List bookmarks (raw API)
curl -s "$LINKDING_URL/api/bookmarks/" \
  -H "Authorization: Token $LINKDING_TOKEN" | jq .

# Create bookmark (raw API)
curl -s -X POST "$LINKDING_URL/api/bookmarks/" \
  -H "Authorization: Token $LINKDING_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"url":"https://example.com","tag_names":["test"]}'
```

## Troubleshooting

### Issue: "Docker not found"

```bash
# Install Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# Log out and back in, then retry
```

### Issue: Port 9090 already in use

```bash
LINKDING_PORT=9091 bash scripts/install.sh
```

### Issue: API token not working

1. Open Linkding UI at http://localhost:9090
2. Log in with admin credentials
3. Go to Settings → Integrations → REST API
4. Generate or copy your token

### Issue: Bookmarks not archiving

Archiving requires the `-plus` image variant:
```bash
bash scripts/install.sh --with-archiving
```
