# SearXNG Search Engine Manager

## What This Does

Deploy your own privacy-respecting search engine in minutes. SearXNG aggregates results from 70+ search engines (Google, Bing, DuckDuckGo, etc.) without tracking you. This skill handles installation, configuration, engine management, and maintenance.

**Example:** "Set up a private search engine on port 8080, enable Google/Bing/Wikipedia, disable tracking, auto-update weekly."

## Quick Start (5 minutes)

### 1. Install with Docker (Recommended)

```bash
bash scripts/install.sh --method docker --port 8080
```

This will:
- Pull the latest SearXNG Docker image
- Generate a secret key
- Create config directory at `~/.config/searxng/`
- Start SearXNG on `http://localhost:8080`

### 2. Install Bare-Metal (No Docker)

```bash
bash scripts/install.sh --method bare-metal --port 8080
```

### 3. Verify Installation

```bash
bash scripts/manage.sh status
# Output:
# ✅ SearXNG is running on http://localhost:8080
# Engines: 42 enabled, 31 disabled
# Uptime: 2m 15s
# Version: 2024.12.28
```

## Core Workflows

### Workflow 1: Quick Deploy with Docker

```bash
# Deploy with defaults (port 8080, all popular engines enabled)
bash scripts/install.sh --method docker --port 8080

# Deploy with custom domain (behind reverse proxy)
bash scripts/install.sh --method docker --port 8080 --base-url https://search.yourdomain.com
```

### Workflow 2: Manage Search Engines

```bash
# List all available engines
bash scripts/manage.sh engines list

# Enable specific engines
bash scripts/manage.sh engines enable google bing wikipedia arxiv

# Disable engines
bash scripts/manage.sh engines disable yahoo aol

# Show enabled engines
bash scripts/manage.sh engines status
# Output:
# ✅ google (general) — weight: 1.0
# ✅ bing (general) — weight: 0.8
# ✅ wikipedia (general) — weight: 0.5
# ✅ arxiv (science) — weight: 0.7
# ❌ yahoo (general) — disabled
```

### Workflow 3: Test a Search

```bash
# Search from CLI
bash scripts/manage.sh search "self-hosted privacy tools"
# Output:
# 1. [Awesome-Selfhosted] https://github.com/awesome-selfhosted/...
# 2. [PrivacyGuides] https://privacyguides.org/...
# 3. ...

# Search specific category
bash scripts/manage.sh search --category science "quantum computing 2026"

# JSON output for scripting
bash scripts/manage.sh search --format json "linux kernel"
```

### Workflow 4: Update SearXNG

```bash
# Update Docker image
bash scripts/manage.sh update

# Output:
# Pulling searxng/searxng:latest...
# ✅ Updated from 2024.12.28 → 2025.01.15
# Restarting container...
# ✅ SearXNG is running on http://localhost:8080

# Auto-update via cron (weekly)
bash scripts/manage.sh auto-update --schedule weekly
```

### Workflow 5: Backup & Restore Configuration

```bash
# Backup config
bash scripts/manage.sh backup
# ✅ Config backed up to ~/.config/searxng/backups/2026-03-05.tar.gz

# Restore config
bash scripts/manage.sh restore ~/.config/searxng/backups/2026-03-05.tar.gz
```

## Configuration

### Config File (~/.config/searxng/settings.yml)

```yaml
general:
  instance_name: "My Search"
  debug: false

search:
  safe_search: 0           # 0=off, 1=moderate, 2=strict
  default_lang: "en"
  autocomplete: "google"   # google, dbpedia, duckduckgo, startpage

server:
  port: 8080
  bind_address: "0.0.0.0"
  secret_key: "<auto-generated>"
  base_url: false           # Set if behind reverse proxy

ui:
  default_theme: simple
  results_on_new_tab: false
  infinite_scroll: true

outgoing:
  request_timeout: 6.0
  max_request_timeout: 15.0
  useragent_suffix: ""
  pool_maxsize: 20

engines:
  - name: google
    engine: google
    shortcut: g
    disabled: false
    weight: 1.0

  - name: bing
    engine: bing
    shortcut: b
    disabled: false
    weight: 0.8

  - name: duckduckgo
    engine: duckduckgo
    shortcut: ddg
    disabled: false

  - name: wikipedia
    engine: wikipedia
    shortcut: wp
    disabled: false
```

### Environment Variables

```bash
# Override config via env
export SEARXNG_PORT=8080
export SEARXNG_BASE_URL="https://search.yourdomain.com"
export SEARXNG_SECRET="your-secret-key"
export SEARXNG_CONFIG_DIR="$HOME/.config/searxng"
```

## Advanced Usage

### Run Behind Nginx Reverse Proxy

```bash
# Generate Nginx config
bash scripts/manage.sh proxy-config nginx --domain search.yourdomain.com
# Output: Nginx config written to /tmp/searxng-nginx.conf
# Copy to /etc/nginx/sites-available/ and enable
```

### Run Behind Caddy

```bash
bash scripts/manage.sh proxy-config caddy --domain search.yourdomain.com
# Output: Caddy config written to /tmp/searxng-Caddyfile
```

### API Access (JSON)

```bash
# Use SearXNG as an API
curl "http://localhost:8080/search?q=test&format=json" | jq '.results[:3]'
```

### Rate Limiting

```bash
# Set rate limits (prevent abuse if public)
bash scripts/manage.sh rate-limit --requests 30 --window 60
```

### Custom Theme

```bash
# List available themes
bash scripts/manage.sh theme list
# simple, oscar, legacy

# Set theme
bash scripts/manage.sh theme set simple
```

## Troubleshooting

### Issue: Container won't start

**Check logs:**
```bash
docker logs searxng 2>&1 | tail -20
```

**Common fix:** Port already in use
```bash
bash scripts/manage.sh stop
bash scripts/install.sh --method docker --port 8888  # Use different port
```

### Issue: No search results

**Fix:** Check if engines are enabled and reachable
```bash
bash scripts/manage.sh engines test
# Tests each engine and reports connectivity
```

### Issue: Slow searches

**Fix:** Reduce timeout and disable slow engines
```bash
bash scripts/manage.sh engines benchmark
# Ranks engines by response time, suggests disabling slow ones
```

### Issue: Docker permission denied

**Fix:**
```bash
sudo usermod -aG docker $USER
newgrp docker
```
