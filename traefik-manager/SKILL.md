# Traefik Reverse Proxy Manager

## What This Does

Installs and configures [Traefik](https://traefik.io/) as a production-ready reverse proxy with automatic SSL via Let's Encrypt and Docker container auto-discovery. Add a few labels to any Docker container and Traefik automatically routes traffic to it — no config file editing needed.

**Example:** "Deploy Traefik, point `app.example.com` to your Docker container with auto-SSL, zero-downtime reloads."

## Quick Start (10 minutes)

### 1. Install Traefik via Docker

```bash
# Create Traefik directories
sudo mkdir -p /opt/traefik
sudo mkdir -p /opt/traefik/config
sudo mkdir -p /opt/traefik/acme

# Set permissions for ACME (Let's Encrypt) storage
sudo touch /opt/traefik/acme/acme.json
sudo chmod 600 /opt/traefik/acme/acme.json

# Create Docker network for Traefik
docker network create traefik-public 2>/dev/null || true
```

### 2. Generate Traefik Configuration

```bash
# Set your email for Let's Encrypt
export TRAEFIK_ACME_EMAIL="you@example.com"

# Generate docker-compose.yml
bash scripts/setup.sh --email "$TRAEFIK_ACME_EMAIL"
```

### 3. Start Traefik

```bash
cd /opt/traefik
docker compose up -d

# Verify it's running
docker compose logs -f --tail=20
```

### 4. Add a Service

```bash
# Example: route app.example.com to a container
bash scripts/add-service.sh \
  --domain app.example.com \
  --container my-app \
  --port 3000
```

## Core Workflows

### Workflow 1: Deploy Traefik with Dashboard

```bash
bash scripts/setup.sh \
  --email admin@example.com \
  --dashboard \
  --dashboard-domain traefik.example.com \
  --dashboard-user admin \
  --dashboard-password supersecret
```

Access the dashboard at `https://traefik.example.com` to monitor routes, services, and middleware.

### Workflow 2: Add Docker Service with Auto-SSL

Any Docker container with the right labels gets auto-discovered:

```yaml
# In your app's docker-compose.yml
services:
  myapp:
    image: myapp:latest
    networks:
      - traefik-public
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.myapp.rule=Host(`app.example.com`)"
      - "traefik.http.routers.myapp.tls.certresolver=letsencrypt"
      - "traefik.http.services.myapp.loadbalancer.server.port=3000"

networks:
  traefik-public:
    external: true
```

Or use the helper script:

```bash
bash scripts/add-service.sh \
  --domain app.example.com \
  --container myapp \
  --port 3000 \
  --ssl
```

### Workflow 3: Add Middleware (Auth, Rate Limit, Headers)

```bash
# Basic auth middleware
bash scripts/add-middleware.sh \
  --name my-auth \
  --type basicauth \
  --users "admin:$(openssl passwd -apr1 mypassword)"

# Rate limiting
bash scripts/add-middleware.sh \
  --name rate-limit \
  --type ratelimit \
  --average 100 \
  --burst 50

# Security headers
bash scripts/add-middleware.sh \
  --name secure-headers \
  --type headers \
  --sts \
  --frame-deny \
  --content-type-nosniff
```

### Workflow 4: Monitor & Troubleshoot

```bash
# Check Traefik status
bash scripts/status.sh

# Output:
# ✅ Traefik: running (v3.x)
# 📊 Routers: 5 active
# 🔐 Certificates: 4 valid, 0 expiring soon
# 🐳 Docker: 8 containers discovered
# ⚠️  Errors: 0

# View access logs
bash scripts/logs.sh --tail 50

# Check certificate status
bash scripts/certs.sh
```

### Workflow 5: Redirect HTTP → HTTPS

Enabled by default in setup. To configure manually:

```bash
bash scripts/setup.sh --email admin@example.com --force-https
```

### Workflow 6: Load Balancing

```yaml
services:
  app1:
    image: myapp:latest
    labels:
      - "traefik.enable=true"
      - "traefik.http.routers.myapp.rule=Host(`app.example.com`)"
      - "traefik.http.services.myapp.loadbalancer.server.port=3000"

  app2:
    image: myapp:latest
    labels:
      - "traefik.enable=true"
      - "traefik.http.services.myapp.loadbalancer.server.port=3000"
```

Traefik automatically load-balances between containers sharing the same service name.

## Configuration

### Environment Variables

```bash
# Required
export TRAEFIK_ACME_EMAIL="admin@example.com"

# Optional
export TRAEFIK_LOG_LEVEL="INFO"          # DEBUG, INFO, WARN, ERROR
export TRAEFIK_DASHBOARD_PORT="8080"     # Dashboard port (internal)
export TRAEFIK_ENTRYPOINT_HTTP="80"      # HTTP entrypoint
export TRAEFIK_ENTRYPOINT_HTTPS="443"    # HTTPS entrypoint
```

### Static Configuration (traefik.yml)

Generated by `scripts/setup.sh`, stored at `/opt/traefik/traefik.yml`:

```yaml
api:
  dashboard: true
  insecure: false

entryPoints:
  web:
    address: ":80"
    http:
      redirections:
        entryPoint:
          to: websecure
          scheme: https
  websecure:
    address: ":443"

providers:
  docker:
    endpoint: "unix:///var/run/docker.sock"
    exposedByDefault: false
    network: traefik-public
  file:
    directory: /etc/traefik/config
    watch: true

certificatesResolvers:
  letsencrypt:
    acme:
      email: admin@example.com
      storage: /acme/acme.json
      httpChallenge:
        entryPoint: web

log:
  level: INFO

accessLog:
  filePath: /var/log/traefik/access.log
  bufferingSize: 100
```

### Dynamic Configuration (File Provider)

Add custom routes without Docker labels in `/opt/traefik/config/`:

```yaml
# /opt/traefik/config/external-services.yml
http:
  routers:
    external-api:
      rule: "Host(`api.example.com`)"
      service: external-api
      tls:
        certResolver: letsencrypt

  services:
    external-api:
      loadBalancer:
        servers:
          - url: "http://192.168.1.100:8080"
```

## Advanced Usage

### Wildcard SSL Certificates (DNS Challenge)

```bash
# For Cloudflare DNS
export CF_API_EMAIL="you@example.com"
export CF_DNS_API_TOKEN="your-cloudflare-token"

bash scripts/setup.sh \
  --email admin@example.com \
  --dns-challenge cloudflare \
  --wildcard "*.example.com"
```

### TCP/UDP Routing (databases, game servers)

```yaml
# Route PostgreSQL traffic
services:
  postgres:
    image: postgres:16
    labels:
      - "traefik.enable=true"
      - "traefik.tcp.routers.postgres.rule=HostSNI(`db.example.com`)"
      - "traefik.tcp.routers.postgres.tls=true"
      - "traefik.tcp.services.postgres.loadbalancer.server.port=5432"
```

### Canary Deployments (Weighted Load Balancing)

```yaml
http:
  services:
    myapp:
      weighted:
        services:
          - name: myapp-v1
            weight: 90
          - name: myapp-v2
            weight: 10
```

### Health Checks

```yaml
http:
  services:
    myapp:
      loadBalancer:
        healthCheck:
          path: /health
          interval: 10s
          timeout: 3s
```

## Troubleshooting

### Issue: "ACME challenge failed"

**Causes:** Port 80 blocked, DNS not pointing to server, firewall rules.

```bash
# Check port 80 is accessible
curl -I http://yourdomain.com

# Check DNS
dig +short yourdomain.com

# Check firewall
sudo ufw status | grep -E "80|443"
```

### Issue: Container not discovered

```bash
# Check container is on traefik-public network
docker inspect <container> | jq '.[0].NetworkSettings.Networks'

# Check labels
docker inspect <container> | jq '.[0].Config.Labels'

# Ensure traefik.enable=true label exists
```

### Issue: 502 Bad Gateway

```bash
# Container port mismatch — verify the port label matches what the app listens on
docker logs <container> | grep -i "listening"

# Check service is healthy
docker exec <container> curl -s localhost:<port>/health
```

### Issue: Dashboard not accessible

```bash
# Check if dashboard is enabled
grep -A2 "dashboard" /opt/traefik/traefik.yml

# If using secure dashboard, check router labels
docker inspect traefik | jq '.[0].Config.Labels' | grep dashboard
```

## Key Principles

1. **Auto-discovery** — Label Docker containers, Traefik finds them automatically
2. **Auto-SSL** — Let's Encrypt certificates provisioned and renewed with zero config
3. **Zero-downtime** — Config changes apply instantly without restart
4. **File provider** — Route to non-Docker services via YAML files
5. **Middleware** — Auth, rate limiting, headers, redirects — all composable

## Dependencies

- `docker` (20.10+) with Docker Compose v2
- `bash` (4.0+)
- `curl`, `openssl` (for helpers)
- Ports 80 and 443 available
