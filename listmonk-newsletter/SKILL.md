# Listmonk Newsletter Manager

## What This Does

Installs and manages [Listmonk](https://listmonk.app), a high-performance self-hosted newsletter and mailing list manager. Create subscriber lists, design email campaigns, set up automated drip sequences, and track open/click analytics — all without paying $50-500/mo for Mailchimp or ConvertKit.

**Example:** "Set up a newsletter for my blog, import 2000 subscribers, and send a welcome campaign."

## Quick Start (10 minutes)

### 1. Install Listmonk

```bash
bash scripts/install.sh
```

This will:
- Check Docker is installed (install if missing)
- Pull Listmonk Docker image
- Set up PostgreSQL database
- Generate config with secure defaults
- Start the service on port 9000

### 2. Access Admin Panel

```bash
# Default credentials (change immediately!)
echo "URL: http://$(hostname -I | awk '{print $1}'):9000"
echo "Username: admin"
echo "Password: $(grep LISTMONK_ADMIN_PASSWORD .env 2>/dev/null | cut -d= -f2)"
```

### 3. Configure SMTP (Required for sending)

```bash
bash scripts/configure-smtp.sh \
  --host smtp.gmail.com \
  --port 587 \
  --user you@gmail.com \
  --password "your-app-password" \
  --from "Your Newsletter <you@gmail.com>"
```

## Core Workflows

### Workflow 1: Set Up a New Newsletter

```bash
# Install + start Listmonk
bash scripts/install.sh

# Configure SMTP
bash scripts/configure-smtp.sh --host smtp.gmail.com --port 587 --user you@gmail.com --password "app-password"

# Create a subscriber list
bash scripts/manage.sh create-list --name "Weekly Digest" --type public --optin double

# Output:
# ✅ List "Weekly Digest" created (ID: 1)
# 📋 Subscribe URL: http://your-server:9000/subscription/form
# 🔗 API endpoint: POST /api/subscribers
```

### Workflow 2: Import Subscribers

```bash
# From CSV (columns: email, name, list_id)
bash scripts/manage.sh import --file subscribers.csv --list "Weekly Digest"

# Output:
# 📥 Importing 2,847 subscribers...
# ✅ 2,831 imported successfully
# ⚠️  16 skipped (invalid email format)
# 📊 Total subscribers in "Weekly Digest": 2,831
```

**CSV format:**
```csv
email,name
alice@example.com,Alice Smith
bob@example.com,Bob Jones
```

### Workflow 3: Send a Campaign

```bash
# Create and send a campaign
bash scripts/manage.sh send-campaign \
  --name "March Newsletter" \
  --subject "What's new this month 🚀" \
  --list "Weekly Digest" \
  --body-file campaign.html \
  --schedule "2026-03-10T09:00:00Z"

# Output:
# 📧 Campaign "March Newsletter" created
# 📋 Recipients: 2,831 (list: Weekly Digest)
# ⏰ Scheduled: Mon Mar 10, 2026 at 9:00 AM UTC
# 🔗 Preview: http://your-server:9000/campaigns/1
```

### Workflow 4: Check Campaign Analytics

```bash
bash scripts/manage.sh stats --campaign "March Newsletter"

# Output:
# 📊 Campaign: March Newsletter
# ├── Sent: 2,831
# ├── Delivered: 2,798 (98.8%)
# ├── Opened: 1,247 (44.1%)
# ├── Clicked: 389 (13.7%)
# ├── Bounced: 33 (1.2%)
# └── Unsubscribed: 8 (0.3%)
```

### Workflow 5: Manage Subscribers via API

```bash
# Add a subscriber
bash scripts/manage.sh add-subscriber \
  --email "newuser@example.com" \
  --name "New User" \
  --list "Weekly Digest"

# Remove a subscriber
bash scripts/manage.sh remove-subscriber --email "olduser@example.com"

# Query subscribers
bash scripts/manage.sh list-subscribers --list "Weekly Digest" --status confirmed --limit 50
```

## Configuration

### Environment Variables (.env)

```bash
# Server
LISTMONK_HOST=0.0.0.0
LISTMONK_PORT=9000
LISTMONK_ADMIN_USER=admin
LISTMONK_ADMIN_PASSWORD=<auto-generated>

# Database
POSTGRES_USER=listmonk
POSTGRES_PASSWORD=<auto-generated>
POSTGRES_DB=listmonk

# SMTP (configure after install)
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_AUTH=login
SMTP_USER=you@gmail.com
SMTP_PASSWORD=your-app-password
SMTP_FROM="Newsletter <you@gmail.com>"
SMTP_TLS=starttls

# Optional: Reverse proxy
LISTMONK_DOMAIN=newsletter.yourdomain.com
```

Edit `docker-compose.override.yml` for custom config:

services:

## Troubleshooting

### Issue: "Docker not found"

```bash
# Install Docker
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# Log out and back in, then retry
```

### Issue: Emails going to spam

**Fixes:**
1. Set up SPF record: `v=spf1 ip4:YOUR_SERVER_IP ~all`
2. Set up DKIM (Listmonk generates keys)
3. Set up DMARC: `v=DMARC1; p=none; rua=mailto:you@example.com`
4. Use a dedicated IP or transactional email service (SES, Postmark)

### Issue: Port 9000 already in use

```bash
# Change port in .env
sed -i 's/LISTMONK_PORT=9000/LISTMONK_PORT=9001/' .env
bash scripts/manage.sh restart
```

### Issue: Database connection errors

```bash
# Check PostgreSQL is running
docker compose ps
# Restart services
bash scripts/manage.sh restart
# Check logs
bash scripts/manage.sh logs --tail 50
```

## Key Principles

1. **Data ownership** — Your subscribers, your server, your data
2. **Cost savings** — Free forever vs $50-500/mo for SaaS alternatives
3. **Performance** — Listmonk sends millions of emails efficiently (Go-based)
4. **Privacy** — No tracking by third parties, GDPR-friendly
5. **API-first** — Full REST API for automation
