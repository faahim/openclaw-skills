# Typesense Search Engine Manager

## What This Does

Installs and manages [Typesense](https://typesense.org), a blazing-fast open-source search engine with typo tolerance, faceting, geo-search, and vector search. Perfect for adding instant search to apps, managing document collections, and running full-text search without Elasticsearch complexity.

**Example:** "Install Typesense, create a products collection with schema, index 10k documents, and query with typo tolerance — all in 5 minutes."

## Quick Start (5 minutes)

### 1. Install Typesense Server

```bash
bash scripts/install.sh
```

This will:
- Detect your OS (Linux amd64/arm64, macOS)
- Download the latest Typesense binary
- Create data directory at `~/.typesense/data`
- Generate a random API key
- Save config to `~/.typesense/config.ini`

### 2. Start the Server

```bash
bash scripts/run.sh start
# Typesense running on http://localhost:8108
# API key saved in ~/.typesense/config.ini
```

### 3. Verify It Works

```bash
bash scripts/run.sh health
# Output: {"ok":true}
```

### 4. Create Your First Collection

```bash
bash scripts/manage.sh create-collection '{
  "name": "products",
  "fields": [
    {"name": "title", "type": "string"},
    {"name": "description", "type": "string"},
    {"name": "price", "type": "float"},
    {"name": "category", "type": "string", "facet": true},
    {"name": "rating", "type": "float"}
  ],
  "default_sorting_field": "rating"
}'
```

### 5. Index Documents

```bash
bash scripts/manage.sh index products '{
  "title": "Wireless Headphones",
  "description": "Noise-cancelling Bluetooth headphones with 30h battery",
  "price": 79.99,
  "category": "Electronics",
  "rating": 4.5
}'
```

### 6. Search with Typo Tolerance

```bash
bash scripts/manage.sh search products "wireles headphon"
# Finds "Wireless Headphones" despite typos!
```

## Core Workflows

### Workflow 1: Manage Collections

```bash
# List all collections
bash scripts/manage.sh list-collections

# Get collection info
bash scripts/manage.sh get-collection products

# Delete a collection
bash scripts/manage.sh drop-collection products
```

### Workflow 2: Bulk Import Documents

```bash
# Import from JSONL file (one JSON per line)
bash scripts/manage.sh bulk-import products /path/to/documents.jsonl

# Import from JSON array file
bash scripts/manage.sh bulk-import-json products /path/to/documents.json
```

### Workflow 3: Advanced Search

```bash
# Search with filters and facets
bash scripts/manage.sh search products "headphones" \
  --filter "category:Electronics && price:<100" \
  --facet "category" \
  --sort "price:asc" \
  --limit 10

# Geo search (if collection has geopoint field)
bash scripts/manage.sh search stores "" \
  --filter "location:(48.85, 2.35, 10 km)"

# Search with grouping
bash scripts/manage.sh search products "phone" \
  --group "category" \
  --group-limit 3
```

### Workflow 4: Manage API Keys

```bash
# Create a search-only key
bash scripts/manage.sh create-key '{"description": "Search-only key", "actions": ["documents:search"], "collections": ["products"]}'

# List all keys
bash scripts/manage.sh list-keys

# Delete a key
bash scripts/manage.sh delete-key <key-id>
```

### Workflow 5: Collection Aliases

```bash
# Create alias (for zero-downtime reindexing)
bash scripts/manage.sh create-alias products products_v2

# List aliases
bash scripts/manage.sh list-aliases
```

### Workflow 6: Export & Backup

```bash
# Export all documents from a collection
bash scripts/manage.sh export products > products-backup.jsonl

# Take a snapshot (full backup)
bash scripts/manage.sh snapshot /path/to/backup/dir
```

## Configuration

### Server Config (~/.typesense/config.ini)

```ini
[server]
api-address = 0.0.0.0
api-port = 8108
data-dir = /home/user/.typesense/data
api-key = <auto-generated>
log-dir = /home/user/.typesense/logs
enable-cors = true
```

### Environment Variables

```bash
# Override defaults
export TYPESENSE_HOST="localhost"
export TYPESENSE_PORT="8108"
export TYPESENSE_API_KEY="your-api-key"
```

## Running as a Service

### Systemd (Linux)

```bash
bash scripts/run.sh install-service
# Creates and enables typesense.service
# Auto-starts on boot

# Control the service
bash scripts/run.sh status
bash scripts/run.sh stop
bash scripts/run.sh restart
bash scripts/run.sh logs
```

### Run in Background (Any OS)

```bash
bash scripts/run.sh start --background
bash scripts/run.sh stop
```

## Troubleshooting

### Issue: "Connection refused" on port 8108

**Check:**
1. Is Typesense running? `bash scripts/run.sh status`
2. Check logs: `bash scripts/run.sh logs`
3. Port in use? `lsof -i :8108`

### Issue: "API key is invalid"

**Fix:** Check your key matches config:
```bash
grep api-key ~/.typesense/config.ini
```

### Issue: High memory usage

**Fix:** Typesense stores index in memory for speed. For large datasets:
```ini
# In config.ini — limit memory
cache-num-entries = 1000
```

### Issue: Slow imports

**Fix:** Use bulk import with JSONL format:
```bash
# Much faster than individual inserts
bash scripts/manage.sh bulk-import collection data.jsonl
```

## Key Principles

1. **Fast** — Sub-10ms search latency, even with typos
2. **Simple** — No JVM, no YAML hell (unlike Elasticsearch)
3. **Typo-tolerant** — Handles misspellings out of the box
4. **Faceted** — Built-in filtering and faceting
5. **Lightweight** — Single binary, ~50MB RAM for small datasets
