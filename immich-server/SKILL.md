# Immich Photo Server Manager

## What This Does

Deploy, manage, and maintain an Immich server — the leading open-source, self-hosted alternative to Google Photos. Features AI-powered search, facial recognition, automatic mobile backup, and timeline browsing. This skill handles the full lifecycle: install, configure, upgrade, backup, and monitor.

**Example:** "Deploy Immich on my server, set up automatic database backups, monitor storage usage, and upgrade when new versions drop."

## Quick Start (10 minutes)

### 1. Check Prerequisites

```bash
# Verify Docker and Docker Compose are installed
docker --version && docker compose version

# If not installed, use the docker-manager skill or:
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
```

### 2. Deploy Immich

```bash
bash scripts/install.sh
```

This will:
- Create `/opt/immich` directory
- Download the latest `docker-compose.yml` and `.env` from Immich's GitHub
- Generate secure database credentials
- Start all containers (server, microservices, machine-learning, redis, postgres)
- Print the access URL

### 3. Access Immich

Open `http://your-server-ip:2283` in a browser. Create your admin account on first visit.

## Configuration

### Environment Variables (.env)

```bash
# Core settings (auto-generated during install)
DB_PASSWORD=<auto-generated>
IMMICH_VERSION=release

# Storage location for photos/videos
UPLOAD_LOCATION=/opt/immich/upload

# Server port
IMMICH_PORT=2283

# Machine learning (disable if low on RAM)
IMMICH_MACHINE_LEARNING_ENABLED=true

# GPU acceleration (optional)
# IMMICH_GPU=cuda  # For NVIDIA
# IMMICH_GPU=openvino  # For Intel
```

### External Access (Reverse Proxy)

```bash
# Generate Nginx config for reverse proxy
bash scripts/proxy-config.sh --domain photos.example.com --ssl

# Output: Nginx config block ready to paste
```

### Storage Configuration

```bash
# Move upload directory to external drive
bash scripts/configure.sh --upload-dir /mnt/external/immich-uploads

# Set storage quota per user (via API)
bash scripts/configure.sh --user-quota admin@example.com 500GB
```

## Advanced Usage

### Automated Backups with Cron

```bash
# Daily backup at 3 AM, keep last 7
echo "0 3 * * * /opt/immich/scripts/backup.sh --keep 7 >> /var/log/immich-backup.log 2>&1" | crontab -

# Or use OpenClaw cron:
# Schedule: { "kind": "cron", "expr": "0 3 * * *" }
# Payload: "Run immich backup: bash /opt/immich/scripts/backup.sh --keep 7"
```

### GPU Acceleration for ML

```bash
# Enable NVIDIA GPU for face detection + CLIP search
bash scripts/configure.sh --gpu nvidia

# Enable Intel GPU (OpenVINO)
bash scripts/configure.sh --gpu intel
```

### API Access

```bash
# Get API key from Immich UI: Account Settings → API Keys

# List all assets
curl -s http://localhost:2283/api/assets \
  -H "x-api-key: YOUR_API_KEY" | jq '.length'

# Get server stats
curl -s http://localhost:2283/api/server/statistics \
  -H "x-api-key: YOUR_API_KEY" | jq .

# Trigger ML jobs (face detection, CLIP indexing)
curl -X POST http://localhost:2283/api/jobs/faceDetection \
  -H "x-api-key: YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"command": "start"}'
```

### Multi-User Setup

```bash
# Create user via API
bash scripts/user.sh --create --email user@example.com --name "Jane" --quota 100GB

# List users
bash scripts/user.sh --list
```

## Troubleshooting

### Issue: Containers won't start

**Check:**
```bash
cd /opt/immich && docker compose logs --tail 50
```

**Common causes:**
- Port 2283 in use: `bash scripts/configure.sh --port 3000`
- Low disk space: `df -h /opt/immich`
- Docker not running: `sudo systemctl start docker`

### Issue: Machine learning container crashes (OOM)

**Fix:** Disable ML or allocate more memory:
```bash
# Option 1: Disable ML (no smart search/face detection)
bash scripts/configure.sh --no-ml

# Option 2: Use smaller ML model
bash scripts/configure.sh --ml-model small
```

### Issue: Slow upload/thumbnail generation

**Check:**
```bash
# View microservices job queue
curl -s http://localhost:2283/api/jobs \
  -H "x-api-key: YOUR_API_KEY" | jq .
```

**Fix:** The queue is processing. Large initial imports can take hours. GPU acceleration helps significantly.

### Issue: Can't connect from mobile app

**Fix:**
1. Ensure port 2283 is open: `sudo ufw allow 2283/tcp`
2. Use your server's LAN IP, not `localhost`
3. For external access, set up reverse proxy with SSL

## Dependencies

- `docker` (20.10+)
- `docker-compose` (v2+)
- `curl` (HTTP requests)
- `jq` (JSON parsing)
- `openssl` (credential generation)
- **Recommended:** 4GB+ RAM (8GB+ with ML enabled)
- **Storage:** Depends on photo library size

## Key Principles

1. **Secure defaults** — Random DB passwords, no exposed ports beyond 2283
2. **Backup before upgrade** — Always snapshot DB before pulling new images
3. **Storage awareness** — Monitor disk usage, alert before full
4. **Non-destructive** — Never delete originals; all operations preserve source files
5. **Privacy-first** — All ML runs locally, no cloud dependency
