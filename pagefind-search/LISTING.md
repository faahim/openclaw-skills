## Tagline

Add instant client-side search to any static site — no server required

## Description

Static sites are fast, cheap, and easy to deploy — but adding search usually means a third-party service or a server. Pagefind changes that completely.

This skill installs Pagefind, indexes your static site's HTML, and generates a lightweight client-side search widget that works entirely in the browser. No backend, no API keys, no monthly fees. Works with Hugo, MkDocs, Jekyll, Astro, Eleventy, or any folder of HTML files.

**What it does:**
- 🔍 Index thousands of HTML pages in seconds
- ⚡ Sub-50ms search results in the browser
- 🎨 Drop-in search UI with CSS theming
- 🏷️ Filter by tags, categories, or custom metadata
- 🌐 Multilingual support (auto-detects page language)
- 🔧 Headless JS API for custom search UIs
- 📦 Tiny bundle — ~200KB for most sites
- 🔄 CI/CD ready — one command in your build pipeline

Perfect for documentation sites, blogs, knowledge bases, and any static site that needs search without server costs.

## Core Capabilities

1. One-command installation — downloads platform-specific binary
2. Static site indexing — scans HTML, builds search index
3. Drop-in search UI — copy-paste snippet adds search bar
4. Selective indexing — control what's searchable with data attributes
5. Filtered search — tag/category filtering out of the box
6. Custom theming — CSS variables for colors, fonts, borders
7. Headless API — build your own search interface
8. Multilingual — indexes by language, supports CJK
9. CI/CD integration — one-liner for GitHub Actions, GitLab CI
10. Zero runtime cost — everything runs client-side

## Installation Time
**5 minutes** — install binary, index site, add HTML snippet
