# Nextcloud Manager

## What This Does

Deploy and manage a self-hosted Nextcloud instance — your own private cloud for files, calendar, contacts, notes, and 300+ apps. This skill handles Docker-based installation, user/group management, app lifecycle, automated backups, and health monitoring. No cloud subscriptions, no data harvesting — everything runs on your server.

**Example:** "Set up Nextcloud with HTTPS, create 5 user accounts, install the Calendar and Tasks apps, and schedule nightly backups to S3."

## Quick Start (10 minutes)

### 1. Check Prerequisites

```bash
bash scripts/nextcloud-manager.sh preflight
```

This checks for Docker, docker-compose, available ports (80/443), and disk space.

### 2. Install Nextcloud

```bash
# Basic install with SQLite (quick start)
bash scripts/nextcloud-manager.sh install --domain localhost --port 8080

# Production install with PostgreSQL + Redis
bash scripts/nextcloud-manager.sh install \
  --domain cloud.example.com \
  --db postgres \
  --cache redis \
  --ssl letsencrypt \
  --admin-user admin \
  --admin-pass "$(openssl rand -base64 16)"
```

### 3. Verify Installation

```bash
bash scripts/nextcloud-manager.sh status

# Output:
# ✅ Nextcloud 29.0.4 running at https://cloud.example.com
# 📦 Database: PostgreSQL 16
# 🔄 Cache: Redis 7.2
# 💾 Storage: 45.2 GB used / 500 GB total
# 👥 Users: 1 (admin)
# 🔒 SSL: Let's Encrypt (expires 2026-05-28)
```

## Core Workflows

### Workflow 1: User Management

```bash
# Create a user
bash scripts/nextcloud-manager.sh user create \
  --username alice \
  --display-name "Alice Smith" \
  --email alice@example.com \
  --quota 10G \
  --groups "staff,engineering"

# List all users
bash scripts/nextcloud-manager.sh user list

# Disable a user (keeps data)
bash scripts/nextcloud-manager.sh user disable --username bob

# Reset password
bash scripts/nextcloud-manager.sh user reset-password --username alice
```

### Workflow 2: App Management

```bash
# List available apps
bash scripts/nextcloud-manager.sh app list --available

# Install apps
bash scripts/nextcloud-manager.sh app install calendar contacts tasks notes deck

# Update all apps
bash scripts/nextcloud-manager.sh app update --all

# Disable an app
bash scripts/nextcloud-manager.sh app disable --name circles
```

### Workflow 3: Automated Backups

```bash
# One-time backup (files + database + config)
bash scripts/nextcloud-manager.sh backup create --output /backups/nextcloud

# Schedule daily backups
bash scripts/nextcloud-manager.sh backup schedule \
  --cron "0 2 * * *" \
  --output /backups/nextcloud \
  --keep 7 \
  --compress

# Backup to S3
bash scripts/nextcloud-manager.sh backup create \
  --output s3://my-bucket/nextcloud-backups \
  --s3-endpoint https://s3.amazonaws.com

# Restore from backup
bash scripts/nextcloud-manager.sh backup restore --from /backups/nextcloud/2026-02-27.tar.gz
```

### Workflow 4: Health Monitoring

## Configuration

### Docker Compose (generated by install)

The install command generates `docker-compose.yml` at `~/.nextcloud/docker-compose.yml`:

```yaml
services:
  nextcloud:
    image: nextcloud:29-apache
    ports:
      - "8080:80"
    volumes:
      - nextcloud_data:/var/www/html
      - ./custom_apps:/var/www/html/custom_apps
      - ./config:/var/www/html/config
    environment:
      - POSTGRES_HOST=db
      - POSTGRES_DB=nextcloud
      - POSTGRES_USER=nextcloud
      - POSTGRES_PASSWORD=${DB_PASSWORD}
      - REDIS_HOST=redis
    depends_on:
      - db
      - redis

  db:
    image: postgres:16-alpine
    volumes:
      - postgres_data:/var/lib/postgresql/data
    environment:
      - POSTGRES_DB=nextcloud
      - POSTGRES_USER=nextcloud
      - POSTGRES_PASSWORD=${DB_PASSWORD}

  redis:
    image: redis:7-alpine
    command: redis-server --requirepass ${REDIS_PASSWORD}

volumes:
  nextcloud_data:
  postgres_data:
```

### Environment Variables

```bash
# Required for S3 backups
export AWS_ACCESS_KEY_ID="your-key"
export AWS_SECRET_ACCESS_KEY="your-secret"

# Optional: custom data directory
export NEXTCLOUD_DATA_DIR="/mnt/storage/nextcloud"
```

## Advanced Usage

### External Storage

```bash
# Mount an S3 bucket as external storage
bash scripts/nextcloud-manager.sh storage add-s3 \
  --name "Company Files" \
  --bucket my-company-files \
  --region us-east-1 \
  --mount-point /CompanyFiles

# Mount a local directory
bash scripts/nextcloud-manager.sh storage add-local \
  --name "Media" \
  --path /mnt/media \
  --mount-point /Media
```

### LDAP/Active Directory

```bash
# Configure LDAP authentication
bash scripts/nextcloud-manager.sh ldap configure \
  --host ldap.example.com \
  --port 636 \
  --base-dn "dc=example,dc=com" \
  --bind-dn "cn=admin,dc=example,dc=com" \
  --bind-password "secret" \
  --user-filter "(objectClass=inetOrgPerson)"
```

### Maintenance Mode

```bash
# Enable maintenance mode (for upgrades/migrations)
bash scripts/nextcloud-manager.sh maintenance on

# Run database maintenance
bash scripts/nextcloud-manager.sh maintenance db-repair

# Rescan files after external changes
bash scripts/nextcloud-manager.sh maintenance files-scan --all

# Disable maintenance mode
bash scripts/nextcloud-manager.sh maintenance off
```

### Upgrade Nextcloud

```bash
# Check for updates
bash scripts/nextcloud-manager.sh upgrade check

# Perform upgrade (auto-backup first)
bash scripts/nextcloud-manager.sh upgrade run --backup-first
```

## Troubleshooting

### Issue: "502 Bad Gateway" after install

**Fix:**
```bash
# Wait 30-60 seconds for first startup, then:
bash scripts/nextcloud-manager.sh health
# If still failing:
bash scripts/nextcloud-manager.sh logs --tail 50
```

### Issue: "Trusted domains" error

**Fix:**
```bash
bash scripts/nextcloud-manager.sh config set \
  --key trusted_domains \
  --value "cloud.example.com,192.168.1.100"
```

### Issue: Upload size limited to 2MB

**Fix:**
```bash
bash scripts/nextcloud-manager.sh tune --upload-limit 16G
```

### Issue: Slow file sync

**Fix:**
```bash
# Enable Redis caching + file locking
bash scripts/nextcloud-manager.sh tune --cache redis --file-locking
```

### Issue: Cron jobs not running

**Fix:**
```bash
# Switch from AJAX to system cron
bash scripts/nextcloud-manager.sh config set --key background_job_mode --value cron
bash scripts/nextcloud-manager.sh cron enable
```

## Key Principles

1. **Docker-first** — Isolated, reproducible, easy upgrades
2. **Production defaults** — PostgreSQL + Redis for real workloads
3. **Backup everything** — Files + DB + config in one command
4. **Health-aware** — Catches issues before users notice
5. **Non-destructive** — Disable users (don't delete), backup before upgrade
